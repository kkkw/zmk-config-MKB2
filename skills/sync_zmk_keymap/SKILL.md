# ZMK キーマップ維持リセット スキル

このスキルは、ZMKのフォークリポジトリで「大元の最新の変更を取り入れたい（または設定をクリーンにしたい）が、自分が苦労して作ったキーマップファイルだけはそのまま維持したい」という時に呼び出します。

## 実行する手順
以下のステップに沿ってユーザーのリポジトリを処理してください。

1. **フォーク元（Upstream）の確認**
   - `git remote -v` を実行し、`upstream` が設定されているか確認する。
   - もし設定されていなければ、ユーザーにフォーク元のURL（または `ユーザー名/リポジトリ名`）を聞き、`git remote add upstream <URL>` で追加する。
2. **キーマップの退避**
   - 現在の `config/MKB.keymap` （または対象のキーマップファイル）を `/tmp/` や `.git/` の外など安全な場所にコピー（バックアップ）する。
3. **リセットの実行**
   - `git fetch upstream` を実行してフォーク元の最新履歴を取得する。
   - `git reset --hard upstream/main` （ブランチ名が違う場合はそれに合わせる）を実行し、現在のリポジトリをフォーク元と完全に同期させる。
4. **キーマップの復元**
   - ステップ2で退避しておいたキーマップファイルを元の場所（`config/`）に上書きコピーして戻す。
5. **コミットとPush**
   - `git add config/MKB.keymap`
   - `git commit -m "Reset to upstream base, retain custom keymap"`
   - `git push -f origin main` を実行して、リセットされた綺麗な履歴をお客様のリモートリポジトリに強制反映する。

## 注意事項
- 実行前に、ユーザーに「未コミットの変更が消えますが、キーマップの退避とリセットを実行してよろしいですか？」と必ず許可を取ること。
- スクロールの値など、一時的に手動で戻す必要がある設定がある場合は退避したキーマップを修正してから復元すること。
