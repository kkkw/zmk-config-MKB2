# BLE スプリット接続トラブルシューティング

## 症状

- 左手を USB に接続するだけなら動く
- 右手を無線（BLE）で接続すると**両手とも固まる**
- 右手のキーが一切反応しない
- 設定リセット（settings_reset）しても改善しない

## 根本原因

`config/west.yml` の ZMK リビジョンが誤ってダウングレードされていた。

| コミット | ZMK revision | 結果 |
|---|---|---|
| `27a2b3e`（2026-07-13） | `v0.3-branch+dya` = `dc5b3d5`（Jun 10） | **正常動作** |
| `76f6271`（2026-07-20）以降 | `7fd6a70`（Feb 3）に誤ダウングレード | 右手接続で両手フリーズ |
| `df05e71`（2026-07-22）以降 | `dc5b3d5b15ccbfac85c167c57fb3ab1485fdf7f0`（Jun 10）に修正 | 復旧 |

`76f6271` の "Pin ZMK 0.3 module revisions" コミットで、te9no が ZMK revision をブランチ名から特定コミットハッシュに固定した際に、最新（June 10）ではなく 4ヶ月前（Feb 3）のコミットを誤って指定した。

## 対処

`config/west.yml` の ZMK revision を June 10 のコミットに固定する：

```yaml
- name: zmk
  remote: te9no
  revision: dc5b3d5b15ccbfac85c167c57fb3ab1485fdf7f0  # Jun 10: Add split input event diagnostics
```

## 診断手順

### 1. 左手だけで動作確認

左手を USB 接続して、キー入力・OLED 表示・LED が正常か確認する。これが動けばファームウェア自体は正常。

### 2. 右手のファームウェア確認

右手を USB 接続して `ls /dev/cu.*` を実行。`cu.usbmodem` ポートが出れば右手ファームは起動している。

### 3. settings_reset を両手に実施

BLE ボンディング情報の汚染が疑われる場合は、左右それぞれに `settings_reset-seeeduino_xiao_ble-zmk.uf2` を書き込んでからメインファームを書き直す。

### 4. 動いていた過去のビルドを探す

GitHub Actions の履歴から動作確認できているビルドを特定し、そのコミットの `west.yml` と現在の差分を比較する。ZMK revision の変更が原因のことが多い。

## このキーボードの設定上の注意点

### 右手（ペリフェラル）

| 設定 | 値 | 理由 |
|---|---|---|
| `CONFIG_ZMK_SLEEP` | `n` | ZMK issue #2904: ペリフェラルがディープスリープするとセントラルが無限ハングする |
| `CONFIG_ZMK_BLE_EXPERIMENTAL_CONN` | 設定しない | ホスト接続用の設定。ペリフェラルはホストに直接つながないので不要かつ干渉の可能性あり |
| `CONFIG_BT_PERIPHERAL_PREF_MAX_INT` | `24`（MIN より大きい値） | セントラルがホストと右手の両方の BLE を同時管理する際にスケジューラの余裕を確保 |
| `CONFIG_BT_BUF_ACL_TX_COUNT` | `20` | トラックボールの HID レポートが BLE バッファを溢れないようにする |

### 左手（セントラル）

| 設定 | 値 | 理由 |
|---|---|---|
| `CONFIG_ZMK_SPLIT_BLE_CENTRAL_SPLIT_RUN_STACK_SIZE` | `3096` | デフォルト（512）は ZMK Studio・RELAY_EVENT・BLE management など多数のモジュールを有効にしているこのキーボードには不足。**ファイル内に同じ設定が重複する場合は後勝ちになるので注意**（以前 768 が末尾にあって 3096 が無効化されていた） |
| `CONFIG_ZMK_SLEEP` | `y` | セントラル側はスリープ有効のまま問題なし |

### ファームウェア書き込み手順

1. リセットボタンを素早く 2 回押してブートローダーモードに入る
2. Mac に USB ドライブとして表示される
3. `.uf2` ファイルをドライブにコピー
4. 自動で再起動する

ブートローダーに入れない・OLED がノイズ表示になっている場合でも、2 回押しでブートローダーに入れることが多い。まずこれを試す。

## 関連する過去の不具合

| 症状 | 原因 | 対処 |
|---|---|---|
| チャタリング（バースト・短い連続入力） | debounce 不足 + BLE キュー溢れ | debounce 増加・BLE バッファ拡大 |
| 有線でも固まる | ZMK issue #2904（ペリフェラルのディープスリープ） | 右手の `CONFIG_ZMK_SLEEP=n` |
| スクロールが動かない・極端に遅い | エンコーダーの `tap-ms` が短すぎる（デフォルト 5ms では積分結果が 0 になる） | `tap-ms=100` を設定する |
| 右手接続で両手フリーズ | ZMK revision の誤ダウングレード（このドキュメントの主題） | west.yml を Jun 10 のコミットに修正 |
